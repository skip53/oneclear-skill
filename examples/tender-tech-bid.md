# Example｜招投标 / 技术标支持

## 业务员原始输入

客户下周投标，需要我们提供技术条款、偏差表和盖章承诺。客户还希望写上保证排放 10mg/Nm3 以内、质保两年。

## 客户原话

“下周就投标了，你们把技术条款和偏差表给我，能不能盖章承诺排放 10 毫克以内、质保两年？”

## Agent 分流判断

招投标 / 技术标阶段。当前 Level 3：可给技术支持准备路径，但不可直接形成盖章承诺。

## 工况档案草稿

- [已知] 客户处于招投标阶段。
- [已知] 客户要求技术条款、偏差表、盖章承诺、排放和质保。
- [真实需求判断] 表层需求是快速拿技术标材料，真实风险是客户要求把排放和寿命写成承诺。
- [异常状态] 存在招投标技术承诺风险和质保/排放承诺风险。
- [风险升级等级] red。
- [合规/承诺边界] 排放、质保、盖章材料必须人工确认，不能由 Agent 自动承诺。
- [商业/采购阶段] 招投标 / 技术标阶段。
- [案件状态] blocked。
- [未知] 完整工况、排放历史、验收方法、质保条款、技术负责人确认。

## 售前推进判断

- [决策链状态] 缺口：需确认技术评估人、采购负责人、最终决策人和招标文件责任人。
- [报价 readiness] 不能正式报价：缺完整工况和技术承诺边界。
- [滤料能力匹配] 可整理风险与能力匹配草稿，但不能输出最终型号。
- [试用/验证边界] 若无既有验证数据，只能列验证条件，不能承诺排放。
- [竞品应对] 如涉及竞品条款，只能列事实差异和偏差，不做攻击性比较。
- [商业价值量化] 技术标材料应区分价格、寿命、排放和质保风险，不承诺降本。
- [内部升级包] 必须升级：技术负责人确认方案边界，销售负责人确认采购流程和时间线，管理层确认盖章/质保/排放承诺风险。
- [人工确认] 未确认前不能对外提交盖章承诺。
- [禁止承诺项] 不保证排放，不保证两年寿命，不直接写入技术协议。
- [下一步行动清单] 销售负责人当天补招标文件、完整工况、验收方法和客户决策链；技术负责人审核技术条款和偏差表；管理层确认盖章风险；目标 Level：Level 5，但当前 blocked。

## 追问清单

### 内部技术版

1. 要完整招标技术条款、验收方法和偏差表模板。
   - 优先级：P0
   - 为什么问：不知道条款就不能判断哪些能响应、哪些必须偏离。

2. 要完整工况和历史排放数据。
   - 优先级：P0
   - 为什么问：排放目标不能在缺工况和验证条件时承诺。

3. 确认盖章材料由谁审批，客户接受哪些偏差。
   - 优先级：P0
   - 为什么问：盖章承诺是合同/招投标风险，必须人工确认。

### 客户转述版

1. 麻烦先发完整招标技术条款、验收方法和偏差表模板。
2. 排放 10mg/Nm3 的验收条件、检测周期和现场历史数据能否一起发一下？
3. 盖章承诺我们需要内部技术和管理层确认，哪些条款客户允许写偏差说明？

## 初步诊断摘要

当前不是普通技术咨询，而是招投标承诺风险场景。Agent 只能整理资料缺口和升级包，不能自动输出盖章承诺、排放保证或质保承诺。

## 禁止承诺项

- 不保证排放 10mg/Nm3。
- 不承诺质保两年。
- 不承诺技术条款可直接盖章。
