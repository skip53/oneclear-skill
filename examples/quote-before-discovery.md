# Example｜报价前置信息不足

## 业务员原始输入

客户说先报个价，袋子规格和数量还没发，现场是水泥厂，但工艺段和温度也没说。

## 客户原话

“你们先给我报个价，我看下大概多少钱。”

## Agent 分流判断

询价 / 报价前置。当前 Level 1：可分流，但不可诊断。

## 工况档案草稿

- [已知] 客户要求报价。
- [已知] 行业可能是水泥。
- [真实需求判断] 表层需求是价格，真实问题可能是采购预算或供应商筛选；当前不能判断技术可行性。
- [异常状态] 报价前置资料不足。
- [风险升级等级] amber：缺规格、数量和工况边界。
- [合规/承诺边界] 不能把预算测算变成技术承诺。
- [商业/采购阶段] 询价 / 报价前置阶段。
- [案件状态] discovery。
- [未知] 规格、数量、工艺段、温度、含湿、粉尘、排放要求、当前材质或目标寿命。
- [当前允许输出] 报价 readiness 缺口、最小追问。
- [当前禁止输出] 正式报价、具体材质、寿命/排放承诺。

## 售前推进判断

- [决策链状态] 未识别：不知道使用者、技术评估人、采购负责人和最终决策人。
- [报价 readiness] 不能正式报价：缺规格、数量、工况边界。
- [滤料能力匹配] 缺工况风险证据。
- [试用/验证边界] 暂不进入试用。
- [竞品应对] 未识别竞品场景。
- [商业价值量化] TCO 信息不足。
- [内部升级包] 暂不需要；若客户要求盖章、质保或排放承诺，则升级。
- [下一步行动清单] 销售负责人报价前补规格、数量、工艺段、温度/含湿/粉尘、排放要求和采购用途；目标 Level：Level 2。

## 追问清单

### 内部技术版

1. 先补滤袋规格和数量。
   - 优先级：P0
   - 为什么问：没有规格和数量不能形成报价基础。

2. 确认这是预算询价、正式采购，还是招投标资料准备。
   - 优先级：P0
   - 为什么问：不同阶段对应的报价边界和承诺风险不同。

3. 补工艺段、温度、含湿、粉尘和排放要求。
   - 优先级：P1
   - 为什么问：这些决定是否只能预算报价，以及能否进入技术判断。

### 客户转述版

1. 方便先发一下滤袋规格和数量吗？没有这两个信息只能做很粗的判断。
2. 这次报价是预算参考、正式采购，还是投标资料需要？
3. 现场是哪个工艺段？温度、含湿、粉尘和排放要求大概是什么？

## 初步诊断摘要

当前是报价前置场景，不是选型完成场景。规格、数量和工况边界缺失时，不能正式报价，也不能顺带承诺寿命、排放或材质方向。

## 禁止承诺项

- 不正式报价。
- 不推荐具体材质。
- 不承诺寿命、排放、质保。
