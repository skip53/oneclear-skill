# Example｜信息不足，无法分流

## 业务员原始输入

客户问这个袋子能不能做，帮忙看看。

## 客户原话

无客户原话。

## Agent 分流判断

信息不足，无法分流。当前 Level 0：无法分流。

## 工况档案草稿

- [未知] 行业。
- [未知] 新项目、旧项目故障还是替换。
- [未知] 工艺段。
- [未知] 温度、含湿、粉尘、除尘器信息。
- [未知] 客户目标。
- [真实需求判断] 无法判断表层需求、真实工况问题和商业目标是否一致。
- [异常状态] 缺客户原话，无法分流。
- [风险升级等级] amber：继续回答会造成误判。
- [合规/承诺边界] 不涉及具体承诺，但禁止报价、材料方向和寿命/排放判断。
- [商业/采购阶段] 商业阶段未知。
- [案件状态] intake。
- [风险] 当前输入无法支持任何技术判断。

## 追问清单

### 内部技术版

1. 先确认这是一条新项目选型、旧项目故障，还是替换现有滤袋。
   - 优先级：P0
   - 为什么问：三类问题的诊断路径完全不同。

2. 要求业务员补客户原话或聊天记录。
   - 优先级：P0
   - 为什么问：只有业务员概括时，很容易把客户真实问题转述错。

3. 至少补行业、工艺段、客户想解决的问题。
   - 优先级：P0
   - 为什么问：没有这些信息不能进入工况诊断。

### 客户转述版

1. 这个现场是新项目要选袋子，还是原来的袋子已经出问题了，或者想替换现有袋子？
2. 客户原话方便转发一下吗？我需要看客户具体怎么说。
3. 现场是什么行业、哪个工艺段？客户最想解决的问题是什么？

## 初步诊断摘要

当前输入无法分流，也不能诊断或选型。下一步只做最小澄清。

异常 / 合规 / 商业阶段：该输入缺客户原话和业务目标，不能进入技术或商业推进。

## 禁止承诺项

- 不做技术判断。
- 不给材料方向。
- 不报价。
