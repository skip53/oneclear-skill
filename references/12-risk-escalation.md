# 12｜风险升级协议

风险标签只是初筛。风险升级决定当前案件是否可以继续推进。
本文件每条触发条件都按「机制 + 损失类型 + 替代路径」组织——读完一条，就知道**这条为什么是 red/no-go**、**触发后会造成什么损失**、**应该走哪条替代推进路径**。

> 通用原则：red = 暂停确定建议，升级人工确认；no-go = 明确不能承诺，给替代路径。两条线都**不绕过**，**不静默**。

---

## 风险等级

| 等级 | 含义 | 允许动作 | 输出要求 |
| --- | --- | --- | --- |
| green | 风险可见且证据基本够用 | 正常输出追问、方向或下一步 | 标准 Round 模板 |
| amber | 有关键未知或需求 gap | 输出假设、证据需求、限制 | 标注 `[假设]`、`[待证]` |
| red | 存在承诺、责任、安环、合规或重大商业风险 | **停止确定建议**，升级人工确认 | red 状态声明 + 触发原因 + 替代路径 |
| no-go | 客户要求与证据、能力或合规边界冲突 | **明确不能承诺**，给替代推进路径 | no-go 状态声明 + 不能承诺的内容 + 可替代路径 |

---

## Red 触发条件（每条含机制 + 损失 + 替代路径）

> Red = 暂停确定建议 + 升级人工确认。**不允许 Agent 在 red 状态下输出方案/承诺/选型。**

### R1. 客户要求保证寿命、保证排放、完全替代

- **机制（为什么会触发）**：保证 = 合同/法律责任。Agent 无现场试用数据 / 无技术评估 → 承诺 = 拿公司信誉对赌。
- **损失类型**：合规处罚 / 客户索赔 / 商业信誉损失。
- **替代路径**：改为"目标记录 + 验证条件清单"（13-compliance-and-commitment.md 的允许话术模板），要求试用结果 + 技术负责人确认后再承诺。

### R2. 故障责任或索赔未查清

- **机制**：索赔 = 法律链路起点。在证据链未建立前归因 = 站队（站客户 → 供应商反诉；站供应商 → 客户反诉）。
- **损失类型**：双向索赔 / 信誉损失。
- **替代路径**：走 11 异常节"责任归因压力"流程，列证据需求（同批 / 工况 / 检测 / 时序），转技术负责人 + 法务。

### R3. 招投标、合同、盖章技术条款需要确定承诺

- **机制**：盖章条款 = 法律文件。无证据承诺 → 合同条款写错 = 后续违约/纠纷。
- **损失类型**：合同违约 / 招投标废标 / 法律责任。
- **替代路径**：所有盖章条款走技术负责人 + 销售负责人 + 法务三签，Agent 只输出"参考技术条款草案" + "未确定项清单"。

### R4. 安环风险：火星、着火、爆炸、环保处罚或安全事故

- **机制**：安环 = 不可逆事件。一次火星烧穿 / 一次粉尘爆炸 = 人身伤害 + 监管处罚 + 永久关停风险。
- **损失类型**：人身伤害 / 监管处罚 / 关停风险 / 刑事责任。
- **替代路径**：走 13 安环流程，列"系统措施"（火星捕集 + 阻火 + 喷淋 + 泄爆），不只靠滤料；转技术负责人 + 安环负责人。

### R5. 当前稳定方案要求无试用全量替换

- **机制**：替换失败 = 不可回退（见 11 异常节"全量替换压力"）。
- **损失类型**：一次性全部损失 / 客户信任崩塌。
- **替代路径**：推分仓 POC / 小批量试用（19-trial-validation-plan.md），设试用边界 + 判定标准 + 回退条件。

### R6. 客户资料或竞品资料涉及保密

- **机制**：评论/外传 = 法律证据（反垄断 / 不正当竞争 / 保密违约）。
- **损失类型**：法律诉讼 / 商业损失。
- **替代路径**：资料归档脱敏 + 不评论竞品 + 回到我方工况；转销售负责人 + 法务。

### R7. 客户或采购要求在工况未明时出具盖章技术协议、寿命保证、排放保证、质保条款或违约赔偿

- **机制**：盖章 + 工况未明 = 拿未知条件签死合同。
- **损失类型**：合同违约 / 法律责任。
- **替代路径**：列必查工況清单（02-work-condition-schema.md），未凑齐前只出"技术协议草案（未确定项标注）"，凑齐后由技术负责人 + 销售负责人 + 法务三签。

### R8. 客户或业务员直接断言上家质量/掺旧料/竞品不行,要求我方站台归因

- **机制**：framing trap——把"陈述"包装成"结论"，诱导 Agent 接受预判（见 11 异常节"责任归因陷阱"）。
- **损失类型**：失实归因 / 双向索赔。
- **替代路径**：不承认不否认 + 列反问证据（同批 / 检测 / 工况变化 / 时序）+ 转人工。

### R9. 客户在信息严重不全(规格/数量/工艺段/温度/含湿/粉尘/排放任一缺)的情况下要求"先报个价"、"先出个方案"或"按我们这周过会先给资料"

- **机制**：信息不全 = 方案/价格必然失真 → 现场不达预期 → 客户反过来告 Agent 失实。
- **损失类型**：失实方案 / 客户信任崩塌 / 后续更换成本。
- **替代路径**：列必查工況清单 + 报价前先做 Level 评估（17-quote-readiness.md），未达 Level 2 不报价；推"分阶段推进"（先补 P0 → 出预算 → 工况齐后出正式报价）。

## No-Go 触发条件（每条含机制 + 损失 + 替代路径）

> No-Go = 明确不能承诺 + 给替代路径。**比 red 更严格——客户要求与证据/能力/合规边界根本冲突，按 red 推进也无解。**

### N1. 客户拒绝提供任何关键工况，却要求确定材质、寿命或排放承诺

- **机制**：无证据 + 强承诺 = 拿现场做实验 + 拿公司信誉对赌。
- **损失类型**：合规处罚 / 双向索赔 / 现场失败。
- **替代路径**：明确说"无工況 = 无承诺"。可给"分阶段推进路径"——先免费做工况评估（客户提供基础参数）→ 再给方向 → 再做小批量试用 → 最后给承诺。**客户拒绝任何阶段配合 = 终止合作**。

### N2. 客户要求隐藏风险或绕过试用

- **机制**：隐藏风险 = 后续现场失败时无法溯源 → 客户反过来告 Agent 推荐失实。
- **损失类型**：法律诉讼 / 商业损失。
- **替代路径**：明确说"风险必须显性记录，绕过试用 = 终止"。不接此类项目。

### N3. 客户要求用低价方案替代稳定方案，但不能接受任何验证

- **机制**：低价 + 不可验证 = 不可能成立（见 04 替换降本风险）。
- **损失类型**：方案失败 / 客户索赔。
- **替代路径**：明确说"低价 + 不可验证 = 商业自相矛盾"。可给"试用替代路径"——小批量试用 + 实测寿命对比 + 验证后再谈替换。**客户拒绝任何验证 = 终止**。

### N4. 客户要求签署超出证据和权限的质保、排放或赔偿承诺

- **机制**：超权限承诺 = Agent 无权签 + 公司无法兜底。
- **损失类型**：合同违约 / 法律责任 / 公司损失。
- **替代路径**：明确说"超权限承诺需走公司审批流程"。Agent 只出"技术评估意见"，盖章承诺由公司授权人签署。**客户要求 Agent 直接签 = 终止**。

## Red / No-Go 输出强制结构

> Red 或 No-Go 触发时，本轮输出**必须**包含以下 6 段。缺一段即视为绕过升级。

```markdown
## 风险等级：<red / no-go>

1. **触发原因**：<从上述 9 条 Red 或 4 条 No-Go 中选 1 条或叠加>
2. **机制（为什么是 red/no-go）**：<损失类型 + 影响范围>
3. **不能承诺的内容**：<具体不能承诺的清单>
4. **可替代推进路径**：<2-3 条具体替代路径>
5. **是否需要技术负责人、销售负责人或管理层确认**：<是/否 + 哪一级>
6. **下一步负责人与时限**：<具体人 + 具体时限>
```

## 升级路径

| 风险等级 | 升级路径 | 时限要求 |
| --- | --- | --- |
| red | Agent → 技术负责人 → 销售负责人 → 管理层（按金额） | 当轮内完成初步升级 |
| no-go | Agent → 销售负责人 + 技术负责人 → 管理层 | 当轮内明确终止/继续决策 |

---

## 跨等级通用检查清单

> 任何等级判断前先过这 4 条。

1. **证据是否齐全**？不齐 = amber（关键未知），不进入 red/no-go 评估。
2. **framing trap 是否已被识别**？客户/业务员的"上家掺旧料/竞品不行"等断言 = 不作为事实。
3. **是否触达合规边界**？涉及寿命/排放/替换/索赔/合同 = 至少 red。
4. **是否有不可调和的需求冲突**？客户拒绝所有替代路径 = 至少 no-go。
