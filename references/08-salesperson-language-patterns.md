# 08｜业务员常见模糊表达

业务员把客户原话压缩成"袋子不耐用"、"压差高"、"换便宜点"等模糊表达时，**90% 已经丢了关键信息**——丢失了时间窗、位置、比例、运行变化。直接按业务员转述做技术判断 = 把业务员压缩后的"症状标签"当"诊断结论"。

本文件每条模糊表达都按「**机制（为什么不能直接归因） + 可执行对策（怎么拆开） + 典型对话**」三段式组织——读完一条，就知道**这句模糊话背后可能藏什么**、**怎么反问才能拆开**、**应该怎么回应业务员**。

> 通用原则：**任何模糊表达触发时，禁止按字面意思归因**——一律进入「反问 → 拆开 → 分类」流程。模糊表达 ≠ 技术判断。

---

## "客户说袋子不耐用"

- **可能对应**：寿命短、破袋、糊袋、排放超标、成本抱怨。

- **机制（为什么不能理解为"材质差"）**：「不耐用」是业务员对客户**多种抱怨**的统称——它可能指向**寿命数字**（用了多久）、**故障形态**（怎么坏）、**经济感受**（太贵 / 换袋太频繁）。**仅凭"不耐用"就推荐新材质 = 把症状标签当根因，方向可能完全错**。例如：客户说"不耐用"实际是"糊袋"——这是清灰/含湿问题，不是材质问题。

- **可执行对策（拆开"不耐用"的 4 个维度）**：
  1. **时长维度**：用了多久？是 3 个月还是 18 个月？→ 决定是"早期失效"还是"正常老化"
  2. **形态维度**：怎么坏？破袋 / 糊袋 / 排放高 / 整体变脆？→ 决定根因方向
  3. **经济维度**：客户真实痛点是寿命数字、太贵、还是换袋太频繁？→ 决定报价/试用/降本路径
  4. **证据维度**：有照片 / 曲线 / 检测报告吗？→ 决定是否进入诊断

- **典型对话**：
  - 业务员：「客户说袋子不耐用，要求换更耐用的。」
  - ✗ 错误回应：「建议升级 PTFE 材质，更耐用。」
  - ✓ 正确回应：「"不耐用"可能对应多种情况。请先发 ① 这批滤袋实际用了多久 ② 怎么坏的（破袋/糊袋/排放高/变脆） ③ 客户真实痛点（寿命数字/价格/换袋频率） ④ 破损照片或糊袋照片。四个维度凑齐后再给方向。」

## "客户说压差高"

- **可能对应**：糊袋、清灰异常、过滤风速偏高、粉尘变化、结露。

- **机制（为什么不能理解为"滤料透气差"）**：压差是系统阻力的**综合读数**——它由"粉尘层 + 清灰残留 + 含湿 + 风速 + 结露"多因叠加，其中至少 4 项与滤料本身无关。**仅凭"压差高"就推"升级覆膜滤料降低阻力"= 把系统问题误投到材质升级，错失真正修复点**（见 `06-failure-diagnosis-playbook.md` 压差高段）。

- **可执行对策（拆开"压差高"的 3 个时间窗）**：
  1. **时序维度**：从什么时候开始高？是新装就高还是运行 N 个月后高？→ 决定"设计问题"vs"运行漂移"
  2. **清灰维度**：清灰后压差降不降？多久回升？→ 决定"清灰无效型"vs"累积型"
  3. **关联维度**：同期温度/含湿/粉尘浓度有无变化？→ 决定是否结露/粉尘性质变化

- **典型对话**：
  - 业务员：「客户说压差高，要求推荐透气性更好的滤料。」
  - ✗ 错误回应：「建议换高透气度 PTFE 覆膜。」
  - ✓ 正确回应：「压差高可能由 5 种原因引起，其中至少 4 种与滤料无关。请先发 ① 压差曲线（连续 7 天、标注清灰时刻） ② 当前清灰压力/周期 ③ 入口温度 + 含湿量 + 露点。三项凑齐再判断是否要动滤料。」

## "客户想换便宜点"

- **可能对应**：稳定方案降本、竞品替换、预算压力、服务不满。

- **机制（为什么不能理解为"只要低价替代"）**：「换便宜点」是客户**商业压力**的表达，**不一定是技术判断**——它可能来自采购预算紧缩、对当前供应商不满、TCO 没算过、或真实发现了同寿命低成本方案。**仅按"换便宜"推进 = 跳到低价方案推荐，没问"为什么想换"和"愿意牺牲什么"**。结果：要么推荐了不该便宜的（不稳定），要么没解决真实痛点（价格不是问题）。

- **可执行对策（拆开"换便宜点"的 3 个真问题）**：
  1. **驱动力维度**：为什么想换？是预算/不满意/TCO 算不过来/有竞品报价？→ 决定是"真降本"还是"换供应商"
  2. **当前基线维度**：当前方案稳定吗？寿命多长？实际成本？→ 决定"降本空间"和"风险"
  3. **可接受度维度**：客户能接受试用边界吗？愿意分仓 POC 吗？→ 决定推进路径（直接换 vs 试用 vs 谈判）

- **典型对话**：
  - 业务员：「客户要便宜 30%，能推荐什么？」
  - ✗ 错误回应：「可以推荐低成本 PPS 替代。」
  - ✓ 正确回应：「"便宜 30%"是否可能 = 取决于当前基线和寿命。请先发 ① 当前方案实际寿命 + 同工况历史运行数据 ② 客户是否接受分仓 POC / 小批量试用 ③ 客户真实痛点（预算/不满意/TCO）。"低 30% + 同寿命"在数据上是否可能，三项凑齐再判断。」

## "客户说排放不行"

- **可能对应**：破袋、安装密封、滤料精度、设备漏风、在线监测异常。

- **机制（为什么不能理解为"必须上更高等级滤料"）**：排放超标有 ≥ 5 个独立根因（破袋、密封泄漏、旁路阀漏、设备漏风、清灰过强），其中仅"滤料精度不足"一项与滤料相关。**只换高精度滤料、不修密封和破袋，排放数据可能纹丝不动**（见 `06-failure-diagnosis-playbook.md` 排放超标段）。更危险：若现场已有破袋，再换更高精度滤料会让客户把"已破袋"误判为"滤料不够好"，延误更换破损袋的时机。

- **可执行对策（拆开"排放不行"的 3 个根因方向）**：
  1. **设备维度**：是否有破袋（检漏仪扫一遍）？旁路阀密封圈？箱体漏风？→ 4 个非滤料根因先排除
  2. **时序维度**：排放超标是突发还是渐进？→ 突发多与破袋/旁路有关，渐进多与滤料堵塞有关
  3. **数据维度**：CEMS 连续曲线？当前滤料是否覆膜？粉尘细度？→ 决定证据等级

- **典型对话**：
  - 业务员：「客户排放超标，要求换更高精度滤料。」
  - ✗ 错误回应：「建议升级到 PTFE 覆膜 + 精度提升一级。」
  - ✓ 正确回应：「排放超标有 ≥ 5 个根因，其中 4 个与滤料无关。请先确认 ① 是否有破袋（检漏仪扫一遍） ② 旁路阀和箱体密封是否漏风 ③ 排放曲线是突发还是渐进。三个都查完，再决定要不要动滤料。」

## "客户让直接报价"

- **可能对应**：已知方案复购、业务员未提供工况、客户只想比价。

- **机制（为什么不能理解为"可以跳过工况"）**：「直接报价」可能是 ① 已知方案的复购（**这种合法**）② 业务员偷懒没问工况（**这种会出大事**）③ 客户被催着出价（**这种违反 17-quote-readiness 报价准备度**）。**仅按"客户要报价"就出价 = 跳过 Level 评估，工况不全时报价必然失真**。一旦客户按错价签合同，后续违约/索赔由报价方承担。

- **可执行对策（拆开"直接报价"的 3 种场景）**：
  1. **场景 A — 复购**：是否同规格/同工况/同批号？→ 走 17-quote-readiness 的"只能预算报价"路径
  2. **场景 B — 偷懒**：业务员是否提供工況？客户是否提供？→ 走 Level 1 评估，工况不全 → 不能报价
  3. **场景 C — 催着出价**：客户为什么急？是否有竞争报价截止日期？→ 走 red 升级（合同前没有工况 = 失真风险）

- **典型对话**：
  - 业务员：「客户让直接报价，越快越好。」
  - ✗ 错误回应：「好的，请提供规格和数量。」
  - ✓ 正确回应：「"直接报价"分 3 种情况。① 已知方案复购？请发规格/材质/批号 → 我走预算报价。② 首次报价？请发 P0 工况（温度/含湿/粉尘/清灰）→ 走 Level 1 评估，工况不全不报价。③ 客户被催着出价？请告知截止日期和催的原因 → 走 red 升级，工况不全不出价。请先确认是哪一种。」

## 跨条目通用检查清单

> 任何模糊表达触发时，先过这 4 条。

1. **业务员转述 vs 客户原话** 是否分离？模糊表达常把两者混在一起——先拆开再判断。
2. **维度是否拆够**？每条模糊表达都有 3-4 个可拆维度——只问一个维度（如"用了多久"）不够。
3. **证据是否到位**？有照片/曲线/检测报告吗？无证据 = 标 `[待证]`，不归因。
4. **是否触达合规边界**？涉及寿命/排放/替换/索赔/合同 → 走 13-compliance-and-commitment.md + 12-risk-escalation.md。
