# 20｜竞品应对

竞品应对的目标不是"用低价打竞品"，而是弄清客户选择标准、当前基线和替换风险。
本文件按「**机制（为什么这种应对有效） + 必问问题（拆开客户真实意图） + 推荐姿态（按场景给方向） + 因果黑名单**」四段式组织——读完一份，就知道**为什么不能直接打价格战**、**怎么问出客户真实选择标准**、**每个场景该怎么应对**、**什么话绝不能对客户说**。

> 通用原则：竞品应对 = **事实 + 边界**——不攻击竞品、不外传资料、不做无依据优劣判断。

---

## 4 种竞品场景（含机制 + 真实问题 + 应对重点）

### 场景 A：竞品稳定但太贵

- **机制（为什么客户会这样想）**：客户发现竞品稳定运行但 TCO 偏高（采购单价 / 寿命 / 维护综合）。**客户真实问题 = 总成本压力，不是"竞品不好"**。
- **真实问题**：客户想降总成本或采购单价，但**不接受牺牲稳定性**。
- **应对重点**：**当前基线 + TCO + 小批量试用**——不说竞品贵，只说"我方产品在客户工况下的 TCO 数据"。

### 场景 B：竞品出现故障

- **机制（为什么客户会换）**：竞品故障让客户重新评估供应商——客户**不一定想换供应商**，可能想要故障原因 + 索赔 + 恢复稳定。
- **真实问题**：客户想恢复稳定或追责，**不一定想替换**。
- **应对重点**：**先做故障诊断和证据链**——先帮客户理清楚根因，再谈替换方向（见 06-failure-diagnosis-playbook.md + 13-compliance-and-commitment.md 索赔段）。

### 场景 C：招投标比竞品

- **机制（为什么这是技术战）**：招投标中客户会逐条对比技术参数、业绩、报价、承诺。**客户选择标准 = 综合最优**（不是单一价格）。
- **真实问题**：客户在比较技术条款和商务风险。
- **应对重点**：**技术偏差 + 承诺边界 + 人工确认**——列出每条偏差项的依据，不夸大、不承诺超出能力的指标。

### 场景 D：客户拿竞品资料压价

- **机制（为什么这是采购谈判）**：客户带竞品资料（出厂报告 / 价格单 / 业绩单）= 谈判筹码，**不是真要换**。
- **真实问题**：采购谈判 — 客户想要"更优价格 + 同等服务"。
- **应对重点**：**不外传 + 不做无依据优劣判断**——资料归档脱敏，回到我方工况与价格体系。

## 必问 5 问（拆开客户真实选择标准）

> 5 问按重要性排序，每问都对应一个隐藏信息——不问到 = 应对方向可能错。

1. **当前竞品实际寿命、压差、排放和维护情况是什么？** → 决定"基线是否真的稳定"+"降本空间"
2. **客户要降的是采购单价，还是总使用成本？** → 决定走"价格谈判"还是"TCO 演示"
3. **客户是否接受分仓或小批量试用？** → 决定走"直接换"还是"试用验证"路径
4. **竞品资料来源是否可用于内部技术判断？** → 决定是否走保密边界（13 竞品资料保密段）
5. **客户最终选择标准：价格、寿命、排放、交期、服务、质保，哪个优先？** → 决定谈判筹码和让步空间

---

## 推荐姿态（按场景给方向）

- **稳定竞品降本**：需**当前基线**（寿命/压差/排放/维护记录）+ **小批量试用**（分仓 POC）
- **竞品故障替换**：先**确认故障归因**（证据链），再谈**替换方向**（基于根因）
- **技术标竞争**：列**偏差**（每条偏差的依据）+ **证据**（同类项目业绩），不攻击竞品
- **采购压价**：列**我方价值**（寿命/服务/交期/质保综合），不**直接对标竞品价格**

---

## 因果黑名单：竞品应对 4 条「绝不要做 X，因为 Y」

> 攻击竞品 = 留下书面证据 + 触发法律风险 + 客户对前后两家都失去信任。**任何一条违反都让竞品应对从"事实"变成"情绪"**。

| # | 绝不要说 | 机制（为什么） | 替代话术 |
|---|---|---|---|
| 1 | 绝不要说"我们这个肯定比竞品好" | 没有现场对比的"肯定好" = 无依据承诺 + 后续现场效果不达预期时客户反索赔 | 改用"在贵司工况下，我方产品有 X 数据 / Y 业绩支撑" |
| 2 | 绝不要说"便宜方案寿命差不多" | "便宜 + 同寿命"在数据上 = 商业自相矛盾（见 04 替换降本风险） | 改用"需要试用数据 + 实测寿命后再判断" |
| 3 | 绝不要说"可以完全替代" | "完全替代" = 拿未知条件签死合同（见 13 招投标合同段） | 改用"试用边界内可验证，全量效果需扩展验证" |
| 4 | 绝不要说"竞品就是材质不行" | 评论竞品 = 留下书面证据 = 触发反垄断/不正当竞争诉讼（见 13 竞品资料保密段） | 改用"我方产品在贵司工况下表现如何"，不评论竞品 |

---

## 跨场景通用检查清单

> 任何竞品应对场景触发时，先过这 4 条。

1. **是否触达竞品评价**？是 → 不评价，回到我方工况。
2. **是否触达客户资料保密**？是 → 归档脱敏，不外传。
3. **是否触达合同/招投标**？是 → Agent 只出草案，不直接出盖章版。
4. **是否触达价格承诺**？是 → 改用"目标 + 验证条件 + 人工确认"模板。
