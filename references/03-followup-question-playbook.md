# 03｜追问清单 Playbook

追问清单不是技术 checklist。它是业务员下一轮可以拿去问客户的最短路径。

## 基本规则

- 一次最多 3-7 个问题。
- 先问会改变判断路径的问题。
- 每个问题必须说明为什么问。
- 输出内部技术版和客户转述版。
- 不把客户听不懂的术语直接抛给业务员。
- 客户转述版必须把技术术语降级为业务/生活语言,且每条不超 2 句,可以直接复制发给客户。
  - 反面案例(不要): 「请确认常态温度和峰值温度,以及含湿量和露点窗口」
  - 正向案例(参照成功案例): 「麻烦问下窑尾烟气平时温度大概多少,最高会冲到多少?」「现场有没有含湿高、启停后结露,或者以前糊袋的情况?」
  - 翻译规则:
    - 常态温度 → 平时温度
    - 峰值温度 → 最高会冲到多少、瞬间最高多少
    - 露点 → 结露温度、什么时候开始凝水
    - 含湿 → 烟气含水、潮气大不大
    - 过滤风速 / 气布比 → 风速、滤袋负荷
    - 排放浓度 / mg/Nm3 → 排放数值、排放要求
    - 覆膜 → 表面覆膜、表面处理
    - 标况工况 → 实际运行条件
  - 不要把 P 编号、规则名、Level、引用规则号写到客户转述版里。

## 优先级

| 优先级 | 定义 | 示例 |
| --- | --- | --- |
| P0 | 缺了不能分流 / 诊断 / 选型 | 这是新项目、旧项目故障，还是替换现有滤袋？ |
| P1 | 缺了可能误判重大风险 | 温度峰值、含湿、结露、故障位置、破损照片 |
| P2 | 缺了影响方案细节 | 克重、后处理、清灰周期、供应商 |
| P3 | 缺了影响商务推进 | 预算、交期、试用范围、采购阶段 |

## 新项目选型追问顺序

1. 行业和工艺段。
2. 常态温度和峰值温度。
3. 含湿和结露。
4. 腐蚀性气体和含氧。
5. 粉尘性质。
6. 除尘器和清灰方式。
7. 排放、寿命、成本目标。

## 旧项目故障追问顺序

1. 具体故障表现。
2. 当前滤料信息。
3. 故障位置和比例。
4. 温度、压差、启停、清灰变化。
5. 工况近期变化。
6. 照片、样品、检测报告、压差曲线。

## 替换 / 降本追问顺序

1. 当前方案是否稳定。
2. 替换目标。
3. 当前材质、寿命、价格、供应商。
4. 客户可接受的试用方式。
5. 竞品样品或历史运行数据。
6. 替换范围。

## 输出格式

```markdown
### 内部技术版

1. 问题：
   - 优先级：
   - 为什么问：

### 客户转述版

1. 可直接发给客户的话：
```

## 示例

### 内部技术版

1. 请确认当前问题是新项目选型、旧项目故障，还是替换现有滤袋。
   - 优先级：P0
   - 为什么问：三类问题追问路径不同，分流不清时继续选型会跑偏。

### 客户转述版

1. 麻烦先确认下，这个现场是新项目要选袋子，还是现在袋子已经出问题了，或者是想替换原来用的袋子？
