# 01｜问题分流规则

Agent 的第一步是分流，不是回答。分流不清时不进入选型。

任务菜单与任务-目标 Level 映射见 `references/28-task-menu.md`。
每轮交互协议见 `references/27-interactive-round-protocol.md`。

## A. 新项目选型

### 识别信号

- 客户新上除尘项目、新建产线、新增设备。
- 客户问某个工况应该用什么滤袋或滤料。
- 没有当前滤袋寿命、故障或替换记录。
- 重点在如何选型、能否适配、能不能达到排放或寿命目标。

### 优先追问

1. 行业和工艺段是什么？
2. 烟气或粉尘的常态温度和峰值温度是多少？
3. 含湿量如何，是否有低温结露或启停结露？
4. 是否有酸性、碱性或腐蚀性气体？
5. 粉尘是否细、粘、油、磨蚀性强，是否有火星或高温颗粒？
6. 除尘器类型、清灰方式、风量、过滤风速是多少？
7. 排放要求和预期寿命是什么？

### 允许输出

- 工况信息缺口。
- 初步风险点。
- 是否可进入 Level 2 或 Level 3。

### 禁止输出

- 工况不清时给确定材质。
- 没有温度和含湿信息时承诺寿命。
- 没有粉尘性质时承诺抗糊袋或抗磨效果。

## B. 旧项目故障

### 识别信号

- 客户说糊袋、破袋、压差高、排放超标、寿命短、掉袋、变硬、腐蚀。
- 现场已经运行。
- 有当前滤袋材质、使用时间或故障表现。
- 重点在为什么坏、是否材料问题、怎么改善。

### 优先追问

1. 故障具体表现是什么？
2. 当前滤料材质、克重、后处理、规格、使用时长是什么？
3. 故障发生位置和比例是什么？
4. 运行温度、波动、启停频率、是否低温结露？
5. 清灰方式、清灰压力、清灰周期、压差变化如何？
6. 入口粉尘或烟气是否近期变化？
7. 是否有现场照片、破损照片、样品、检测报告或压差曲线？

### 允许输出

- 故障归因假设。
- 证据需求。
- 暂不能下结论的原因。

### 禁止输出

- 直接把故障归因于材质差。
- 没有照片、样品或运行记录时承诺换材料可解决。
- 故障未定位时推荐替换降本方案。

## C. 替换 / 降本

### 识别信号

- 客户已有材质或供应商。
- 客户想换便宜点、寿命更长、交期更稳、服务更好。
- 客户提到竞品价格、竞品材质、原供应商问题。
- 重点在能否替代、替代风险、如何试用。

### 优先追问

1. 当前滤料材质、规格、后处理、寿命、价格区间是什么？
2. 替换目标是降本、延寿、改善排放、解决故障，还是交期或服务问题？
3. 当前方案是否稳定？如果稳定，为什么要换？
4. 客户能接受小批量试用或分仓测试吗？
5. 是否有竞品样品、检测指标、历史运行数据？
6. 替换范围是全量替换还是局部试用？

### 允许输出

- 替换动机判断。
- 替换风险。
- 建议试用路径。

### 禁止输出

- 替换动机不清时承诺降本。
- 不知道当前方案是否稳定时承诺同等寿命。
- 没有试用边界时承诺全量替代。

## D. 信息不足，无法分流

### 识别信号

- 只有“客户问这个怎么搞”“能不能报个价”“帮忙看看”。
- 缺客户原话。
- 缺行业、工况、当前状态或目标。
- 看不出是新项目、故障还是替换。

### 优先追问

1. 这是新项目选型、旧项目故障，还是替换现有滤袋？
2. 有没有客户原话、聊天记录、邮件或会议纪要？
3. 现场属于什么行业和工艺段？
4. 客户现在最想解决的问题是什么：选型、故障、寿命、排放、降本，还是交期？
5. 如果是旧项目，现场现在的具体表现是什么？

### 允许输出

- 3-5 个最小澄清问题。
- 当前不能回答的原因。

### 禁止输出

- 技术方案。
- 选型方向。
- 具体材质。
- 报价建议。

## E. 报价准备

### 识别信号

- 客户/采购/业务员主动问"报个价""多少钱""能不能做"。
- 询价或询比价邮件。
- 还没完成规格、数量、工况确认就催报价。
- 项目处于询价/报价前置/预算阶段。

### 优先追问

1. 滤袋规格（直径、长度、结构）与数量。
2. 配套笼骨、缝制、包装、交付要求。
3. 行业与工艺段、常态温度与峰值温度。
4. 含湿、粉尘性质、腐蚀成分、排放要求。
5. 当前/目标（新项目/故障表现/替换基线/试用目标）。
6. 交期、采购流程、是否招投标。
7. 是否已有指定品牌、目标价位或竞品报价。

### 允许输出

- 报价 readiness 三档判断。
- 报价前置资料缺口清单。
- 预算区间（仅在 Level 3+，且显式标"内部测算"）。

### 禁止输出

- 在规格与工况边界未完整前出正式报价。
- 寿命、排放、质保、全替代承诺。
- 跳人工确认的承诺。

## F. 试用验证

### 识别信号

- 客户接受"先试一批看看"。
- 业务员主动提出分仓、对照、PO 试单。
- 项目处于试用验证阶段。

### 优先追问

1. 试用范围（分仓/同分室不同料/同型号不同供应商）。
2. 对照基线（当前材质/寿命/压差/排放）。
3. 验收指标（寿命/压差/排放/破损率，量化）。
4. 试用周期（月度/季度/半年）。
5. 风险边界（谁承担试错/不达标退出条件）。
6. 商业衔接（试用成功 → 正式订单路径）。

### 允许输出

- 试用边界（范围、指标、周期、退出条件）。
- 对照基线核对清单。
- 试用验证计划草案。

### 禁止输出

- 试用必成承诺。
- 试用期内零破损承诺。
- 没有验收指标的"先试试"承诺。

## G. 招投标支持

### 识别信号

- 客户发来招标文件、技术规格书、偏差表。
- 业务员问"这个标能不能投""技术条款怎么应答"。
- 涉及盖章、投标保证金、合同条款。

### 优先追问

1. 招标技术条款逐条（参数、范围、废标项）。
2. 商务条款与评分标准。
3. 评审节点（技术评审/商务评审/最终答辩）。
4. 历史投标记录与历史客户关系。
5. 是否首次投标此客户/项目。
6. 竞争对手在投标的情况（若知）。
7. 技术承诺边界（盖章文件、不可偏离条款）。

### 允许输出

- 招标条款审查与风险标注。
- 偏差表与应对建议。
- 技术应答要点。
- 人工复核任务清单。

### 禁止输出

- 在无人工确认下擅自承诺技术条款。
- 偏离招标底线条款的应答。
- 任何盖技术章的承诺。

## H. 竞品对比

### 识别信号

- 客户带竞品样品、检测报告、PPT 或报价。
- 客户说"竞品才 X 价""竞品寿命更长"。
- 业务员问"能不能用我们的料替代 XX"。

### 优先追问

1. 竞品型号/材质/克重/后处理。
2. 当前基线（我方运行数据/竞品运行数据）。
3. 对比维度（寿命/压差/排放/价格/交期/服务，权重）。
4. 客户选择标准与权重。
5. 替代范围（全量/部分）。
6. 客户是否提供竞品资料（若提供，启动保密边界）。
7. 我方相对竞品的差异化（若明确）。

### 允许输出

- 对比维度核对清单。
- 我方 vs 竞品的对比维度评估（基于事实，不评价优劣）。
- 试用边界与替代路径。

### 禁止输出

- 评价竞品优劣（公开市场上不作贬低/吹捧）。
- 引用竞品客户数据（保密边界）。
- 承诺替代率（无试用边界时）。

## I. 复购/补单

### 识别信号

- 老客户续单、补单、追加订单。
- 业务员问"这个客户去年下过单，今年又来了"。

### 优先追问

1. 上次订单规格与历史表现。
2. 本次数量、交期、包装。
3. 是否有规格/工况/排放变化。
4. 是否需要特殊检测报告。
5. 商务条款（账期、付款、发票）。

### 允许输出

- 规格核对清单。
- 历史表现回顾。
- 风险变更点（如有）。

### 禁止输出

- 承诺旧合同条款沿用（合同条款需商务评审）。
- 擅自调整技术参数。
