# 14｜商业与采购阶段

售前技术支持不能只看工况，还要判断客户处在哪个商业阶段。商业阶段决定输出深度和下一步动作。

## 阶段识别

| 阶段 | 触发信号 | 技术支持重点 |
| --- | --- | --- |
| 询价 / 报价前置 | 多少钱、报个价、能不能做 | 补规格、数量、材质、工况，不先报价 |
| 技术发现 / 新项目选型 | 新项目、新建、新上、设计院 | 工况参数、排放目标、寿命目标、除尘器条件 |
| 故障索赔 / 责任判断 | 是不是质量问题、赔偿、半年坏 | 证据链、故障归因、不能直接认责 |
| 竞品替换 / 降本 | 竞品太贵、换便宜、同寿命 | 当前基线、替换动机、试用边界 |
| 试用验证 | 小批量、分仓、试用 | 试用范围、指标、周期、验收标准 |
| 招投标 / 技术标 | 招标、投标、技术条款、偏差表 | 技术条款审查、承诺边界、人工确认 |
| 复购 / 交付推进 | 老客户、补单、交期 | 规格确认、历史表现、交付风险 |

## 输出要求

每次输出应标注：

- 当前商业阶段。
- 这一阶段最重要的技术动作。
- 销售下一步要拿到的信息。
- 哪些内容不能在当前阶段承诺。

## 推进原则

- 报价前必须有足够规格和工况边界。
- 故障索赔前必须有证据链。
- 替换降本前必须有当前稳定基线。
- 试用前必须有验收指标。
- 招投标前必须人工确认技术承诺。
