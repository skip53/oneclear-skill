# 19｜工业试用验证计划

工业滤袋试用不是软件 Demo。试用必须控制仓室、数量、基线、周期、记录责任和验收指标，否则试用结果无法解释。
本文件按「**机制（为什么必须控制变量） + 步骤化对策（怎么控制） + 因果黑名单（绝不要 X，因为 Y）**」组织——读完一份，就知道**为什么试用不能"装上看效果"**、**怎么控制变量才有可信结果**、**什么话不能对客户说**。

> 通用原则：试用 = **小规模受控实验**——任何缺对照基线、缺边界、缺记录的"试用"都不能作为选型 / 替换 / 索赔依据。

---

## 适用场景

- 客户要求小批量、分仓或对照试用。
- 稳定竞品方案要求降本替换。
- 故障原因尚不完整，但客户希望验证改进方向。
- 招投标或大项目需要技术负责人确认前的验证依据。

---

## 3 种试用方式（含机制 + 适用条件 + 必查证据）

### 方式 1：分仓对照

- **机制（为什么用分仓）**：在**多仓室袋式除尘器**中保留 1 个仓用原方案（对照仓），其余仓用新方案（试用仓）。**两侧同时运行、同时受同样工况** = 唯一能排除"工况漂移"影响的对照方式。
- **适用条件**：多仓室（≥ 2 仓），可保留原方案对比，客户接受不完全替换。
- **必查证据**：
  - **负荷一致性**：两侧风量、粉尘浓度、温度、含湿——必须**逐日记录**确认无显著差异
  - **记录频率**：压差 / 排放 / 清灰参数至少每日记录
  - **记录周期**：覆盖客户运行的所有典型阶段（高负荷 / 低负荷 / 启停 / 季节变化）

### 方式 2：小批量试用

- **机制（为什么用小批量）**：客户不接受分仓、或风险高（安环/索赔相关）时，用**少量袋**（如 50-200 条）在**指定位置**安装试用。**单边试验 + 短期观察** = 风险可控，但不能完全等同于全量效果。
- **适用条件**：单仓 / 客户不接受分仓 / 风险高 / 时间紧。
- **必查证据**：
  - **数量 + 安装位置**：具体到哪一室、哪一区（迎风面/背风面/袋口附近）
  - **运行边界**：温度/含湿/风速/粉尘负荷的可接受范围
  - **退出条件**：压差/排放/异常升高时的停止试用阈值

### 方式 3：样品 / 实验室验证

- **机制（为什么用样品）**：无法马上进现场（合同未签 / 现场不具备条件 / 时间紧）时，用**实验室加速老化 + 现场样品检测**。**实验室数据 ≠ 现场寿命**——只能作为材料筛选，不作为最终承诺。
- **适用条件**：无法现场试用 / 材料预筛选 / 故障反查。
- **必查证据**：
  - **样品来源**：客户现场取样 / 我方库存 / 第三方提供
  - **检测项目**：抗拉强度 / 透气度 / 耐温 / 耐酸碱 / 红外光谱 / DSC
  - **报告用途**：仅作内部技术判断，不作为对外承诺依据

---

## 试用必备字段

- **试用范围**：仓室、数量、安装位置、安装时间。
- **对照基线**：当前方案、寿命、压差、排放、清灰参数、历史故障。
- **观察周期**：至少覆盖客户关心的运行阶段；不能把短期稳定等同于寿命承诺。
- **验收指标**：压差趋势、排放记录、袋面状态、破损/糊袋情况、清灰稳定性、异常记录。
- **双方责任**：客户记录什么，销售跟进什么，技术负责人复核什么。
- **退出条件**：异常升高、排放风险、安环风险或工况偏离时如何处理。

---

## 因果黑名单：试用 5 条「绝不要做 X，因为 Y」

> 试用边界违反 = 试用结果失真 = 后续选型/替换/索赔依据崩塌。**任何一条违反都让试用结果不能作为决策依据**。

| # | 绝不要做 | 机制（为什么） | 替代动作 |
|---|---|---|---|
| 1 | 绝不要把试用前判断说成全量替换效果 | 试用 ≠ 全量效果——单仓 / 短期 / 受控 ≠ 全仓 / 长期 / 实际 | 改用"试用边界内效果"+ 注明"全量效果需扩展验证" |
| 2 | 绝不用短期试用结果承诺完整寿命 | 短期稳定 ≠ 长期寿命——工况漂移、季节变化、负荷变化都会在 N 个月后显现 | 改用"短期观察"措辞 + 标注"完整寿命需 ≥ 1 个完整运行周期" |
| 3 | 绝不要承诺"低价方案完全替代竞品" | 低价 + 不可验证 = 商业自相矛盾（见 04 替换降本风险） | 改用"试用对比"路径 + 实测寿命数据后再谈替换 |
| 4 | 绝不在缺少对照基线时宣称试用成功 | 无对照 = 不知是试用方案好还是工况变了 | 必须有原方案 / 同期数据 / 同期工况作为对照 |
| 5 | 绝不在试用边界外推广结果到全量 | 边界外推 = 把异常数据当普遍规律 | 试用结果仅适用"相同工况 + 相同运行条件"；工况变化时需重新验证 |

---

## 试用结果不通过时的处理

- **结果不通过时**：不允许直接说"试用失败"——回到根因分析（工况漂移？安装问题？操作问题？材质问题？），列证据链。
- **结果不通过仍要求全量替换时**：走 red 升级（red 触发条件 #5：稳定方案无试用全量替换）。
- **试用期间发生安环事件**：立刻停止试用 → 走 13-compliance-and-commitment.md 安环段。
