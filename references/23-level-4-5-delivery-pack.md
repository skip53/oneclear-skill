# 23｜Level 4 / Level 5 交付包

Level 4 和 Level 5 是售前推进输出，不是最终合同承诺。二者都必须保留人工确认和禁止承诺项。
本文件按「**机制（为什么必须留人工确认） + 必含字段 + 允许/禁止输出（带因果） + Readiness 不通过时的处理**」组织——读完一份，就知道**为什么 Level 4/5 不能跳过人工**、**每个 Level 必含什么**、**什么能说什么不能说**、**缺字段时该怎么处理**。

> 通用原则：Level 4 = 候选方向，Level 5 = 技术支持包。两者都是**人工决策的输入**，不是**自动决策的输出**。所有"候选方向"必须由人最终确认；所有"技术支持包"必须有人工签收。

---

## Level 4｜候选方向包

### 机制（为什么必须留人工确认）

Level 4 输出**直接进入客户的选型决策**——客户会基于候选方向要求报价、试用、签合同。**Agent 自动选出的"候选"一旦方向错 = 客户基于错方向走完选型流程 = 项目失败 + 信誉损失**。**人工确认 = 把"Agent 推荐的"和"客户决策的"分开——Agent 错了可以修订，客户基于错方向走完的流程不可逆**。

### 必含字段

- 当前 Level 和**人工确认提示**（明确标注"以下候选方向需人工技术负责人确认"）
- 客户需求摘要
- 工况档案关键事实
- 真实需求判断
- **1-3 个候选方向**
- **每个候选方向的适用条件、风险、证据要求**
- **试用 / 验证边界**
- 禁止承诺项
- 下一步行动清单

### 允许输出（带因果）

| 允许 | 因为 |
|---|---|
| 候选方向（材质/工艺大类，不写具体型号） | 具体型号 = 合同标的 = 必须人工确认 |
| 验证方法 | 客户可基于验证方法判断方向 |
| 试用范围 | 试用是受控实验 = 风险可控 |
| 证据缺口 | 缺口透明 = 客户决策有依据 |

### 禁止输出（带因果）

| 绝不要做 | 机制（为什么） |
|---|---|
| 绝不要写最终型号 | 最终型号 = 合同标的 = Agent 无权拍板 |
| 绝不要写寿命保证 | 寿命保证 = 法律责任 = 必须人工 + 三签（见 13 质保寿命段） |
| 绝不要写排放保证 | 排放保证 = 法律责任 = 必须人工 + 三签（见 13 环保排放段） |
| 绝不要写质保承诺 | 质保承诺 = 合同条款 = 必须人工 + 三签（见 13 招投标合同段） |
| 绝不要写"完全替代竞品" | "完全替代" = 拿未知条件签死合同 = 触发反索赔（见 13 + 31-blacklist #9） |

## Level 5｜技术支持包

### 机制（为什么必须留人工确认）

Level 5 = **客户选型决策的最终输入**——客户会基于技术支持包做最后决策（签合同 / 拒绝 / 走招投标）。**Agent 自动输出的"支持包"一旦错 = 客户签错合同 = 不可逆**。**人工签收 = 责任落到人——技术负责人对内容负责，销售负责人对商务负责**。

### 必含字段

- 客户需求摘要
- 工况档案
- 真实需求判断
- 风险假设和风险升级等级
- 滤料能力匹配或候选方向
- 试用验证计划
- 客户话术
- 禁止承诺项
- 下一步行动清单：负责人、需补资料、时间点、推进目标、目标 Level
- **人工确认记录或明确的待确认项**

### 允许输出（带因果）

| 允许 | 因为 |
|---|---|
| 客户需求摘要 | 客户决策的回溯依据 |
| 工况档案 | 决策的事实基础 |
| 风险假设 | 风险透明化 = 客户决策有依据 |
| 候选方向 | 客户决策的选项 |
| 试用计划 | 验证路径 = 风险可控 |
| 客户话术 | 业务员可直接使用 |
| 禁止承诺项 | 边界明确 = 保护双方 |
| 下一步行动清单 | 可执行 |

### 禁止输出（带因果）

| 绝不要做 | 机制（为什么） |
|---|---|
| 绝不要写盖章条款 | 盖章 = 法律文件 = Agent 无权盖章（见 13 招投标合同段） |
| 绝不要写具体单价 / 总价 | 价格 = 商务条款 = 必须人工 + 三签 |
| 绝不要写"一定能用 X 年" | "一定" = 法律承诺 = 必须人工 + 三签（见 13 质保寿命段） |
| 绝不要承诺替代效果 | 替代效果 = 拿未知条件签死合同 = 反索赔风险 |
| 绝不要把 Agent 推断当成"已确认事实" | Agent 推断 = 待证假设 = 客户基于此决策 = 风险转嫁 |

## Readiness 不通过时的处理

### Level 4 Readiness 不通过

如果缺以下任一字段，**不能输出 Level 4 候选方向包**：

- 证据要求
- 试用/验证边界
- 人工确认提示

**处理方式**：停在 Round N+1 反问补齐，或降级到 Level 3（仅给选型方向）输出。

### Level 5 Readiness 不通过

如果缺以下任一字段，**不能输出 Level 5 技术支持包**：

- 客户需求摘要
- 工况档案
- 风险假设
- 试用验证计划
- 客户话术
- 禁止承诺项
- 下一步行动清单

**处理方式**：停在 Round N+1 反问补齐，或降级到 Level 4（候选方向包）输出。

## 跨 Level 通用检查清单

> 任何 Level 4/5 输出前，先过这 5 条。

1. **人工确认提示是否在显眼位置**？是 → 通过；否 → 必须补，否则不能输出。
2. **必含字段是否齐全**？是 → 通过；缺任一 → 不能输出。
3. **是否触达合规边界**？是 → 走 13-compliance-and-commitment.md + 12-risk-escalation.md。
4. **是否含"一定/保证/完全替代"等绝对话术**？是 → 改用"目标 + 验证条件 + 人工确认"模板。
5. **是否把 Agent 推断标成"已确认事实"**？是 → 改用"假设/待证"标签，让客户知道这是 Agent 推断。
