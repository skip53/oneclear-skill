# 28｜任务菜单 (Task Menu)

本文件定义 Round 0 使用的任务清单,以及每个任务对应的 P1 默认追问集、目标 Level、典型收敛节奏。

## 任务菜单 (Round 0 输出)

```text
你今天需要处理哪个任务? (选一个最贴的,后面可以再调)

A) 新项目选型 — 客户新建项目/新产线,需要确定滤袋
B) 旧项目故障 — 现场滤袋出问题,要找原因/挽救
C) 替换/降本 — 客户已有方案,想换供应商或降本
D) 报价准备 — 客户要报价/询价,需要准备报价资料
E) 试用验证 — 小批量试用、分仓测试、POC 验证
F) 招投标支持 — 招标/投标技术响应、偏差表
G) 竞品对比 — 客户带竞品资料或压价,要应对
H) 复购/补单 — 老客户补单,需要规格/交期确认
I) 我有客户原话/聊天记录,需要整理 → (选此项请直接粘贴原文)
J) 其他 / 不确定
```

## AskUserQuestion 默认任务确认(条件性)

A-J 菜单保留为自由文本 markdown(10 选项超 AskUserQuestion 单题 4 选项上限)。

**当且仅当默认任务明显时**,在 A-J 菜单 markdown 块之后追加一次 AskUserQuestion 单题单选,提供方向键选择体验:

| 字段 | 值 |
|---|---|
| header chip | `任务确认` |
| 类型 | 单选 |
| 选项 | `确认默认: <推荐任务>` / `调整为: <备选 1>` / `调整为: <备选 2>` / (自动) `其他(可粘原文)` |

**触发条件**: 用户粘贴了内容 / 第一句话任务明显 / `classify_case_type()` 高置信度匹配。
**不触发**: 纯冷启动 `/oneclear` 无任何内容 / 第一句话完全不明确 / J 是默认。

详细实现见 `references/30-askuserquestion-integration.md` 和 `references/27-interactive-round-protocol.md` 的 Round 0 节。

## 任务-目标 Level 映射

| 任务 | 默认目标 Level | 典型收敛轮数 | 收敛后输出形态 |
|---|---|---|---|
| A 新项目选型 | Level 3(选型方向)/ 4(候选方案需人工确认) | 2-4 轮 | 工况档案 + 追问 + 诊断 + 红线;Level 4 加售前推进 |
| B 旧项目故障 | Level 2(归因假设)/ 3(方向)/ 4(候选方案) | 3-5 轮 | 同上 |
| C 替换/降本 | Level 3 / 4 | 3-5 轮 | 加: 当前基线、试用边界、降本商业价值 |
| D 报价准备 | Level 3(预算报价) / 4(正式报价前人工复核) | 2-3 轮 | 加: 报价 readiness 三档判断、报价前置资料缺口 |
| E 试用验证 | Level 4(试用边界) | 3-5 轮 | 加: 试用范围、对照基线、验收指标、周期 |
| F 招投标支持 | Level 5(技术支持包) | 4-6 轮 | 加: 招标条款审查、偏差表、人工确认状态 |
| G 竞品对比 | Level 4(对比方案) | 3-5 轮 | 加: 对比维度、基线、试用边界、保密边界 |
| H 复购/补单 | Level 2-3(规格确认) | 1-2 轮 | 工况档案确认 + 红线,不需候选方案 |
| I 有客户原话需整理 | 取决于内容性质 | 1 轮评估 + N 轮补全 | 由内容决定 |
| J 其他/不确定 | Level 0(分流) | 0 轮直出 → 回到 A-H | 至少回到分流 |

> 客户/业务员如有"想现在就出方案"的压力,任务目标 Level 是底线,不允许为了赶节奏越过。

## 任务-默认 P1 追问集

### A 新项目选型

- 工艺段
- 常态温度 / 峰值温度
- 含湿量 / 是否结露
- 酸碱/腐蚀气体成分
- 粉尘性质(类型、细度、磨蚀性、粘性)
- 除尘器类型 / 清灰方式
- 风量 / 过滤风速
- 入口粉尘浓度
- 排放要求
- 目标寿命 / 目标成本
- 是否有指定品牌或预算区间

### B 旧项目故障

- 故障表现(糊袋/破袋/压差高/排放超标/寿命短)
- 当前滤料材质 / 型号 / 克重 / 后处理
- 已使用时长 / 历史寿命
- 故障位置(袋口/袋身/袋底/缝线) / 比例
- 常态温度 / 含湿量 / 启停频率
- 清灰参数(压力/周期/喷吹方式)
- 压差曲线
- 近期工况变化(原料/燃料/工艺)
- 现场照片/样品/检测报告
- 是否已经在追责或索赔

### C 替换/降本

- 当前材质 / 供应商 / 价格区间
- 当前寿命 / 当前方案是否稳定
- 替换目标(降本比例 / 延寿 / 改善排放 / 改善服务 / 交期)
- 是否愿意小批量试用或分仓测试
- 替换范围(全量 / 部分分室 / 同型号不同供应商)
- 竞品型号 / 报价 / 现有检测数据
- 决策链(技术 / 采购 / 老板)

### D 报价准备

- 规格(滤袋尺寸 / 结构 / 数量)
- 配套笼骨 / 缝制特殊要求
- 工况(行业 / 工艺段 / 温度 / 含湿 / 粉尘 / 排放要求)
- 当前/目标(新项目 / 故障表现 / 替换基线 / 试用目标)
- 商务边界(数量 / 交期 / 采购流程 / 预算 / 是否招投标)
- 技术承诺边界(寿命 / 排放 / 质保要求 / 竞品完全替代)

### E 试用验证

- 试用范围(分仓 / 同型号 / 同分室不同料)
- 对照基线(当前材质 / 寿命 / 压差 / 排放)
- 验收指标(寿命 / 压差 / 排放 / 破损率)
- 试用周期(月度 / 季度 / 半年)
- 风险边界(谁承担试错 / 不达标退出条件)
- 商业衔接(试用成功 → 正式订单路径)

### F 招投标支持

- 招标技术条款 / 偏差表
- 商务条款 / 评分标准 / 废标项
- 评审节点(技术评审 / 商务评审 / 最终答辩)
- 风险边界(技术承诺 / 盖章文件 / 投标保证金)
- 历史投标记录(若有)
- 是否首次投标此项目
- 竞争对手在投标的情况(若知)

### G 竞品对比

- 竞品型号 / 材质 / 克重 / 后处理
- 当前基线(我方运行数据 / 竞品运行数据)
- 对比维度(寿命 / 压差 / 排放 / 价格 / 交期 / 服务)
- 客户选择标准(权重)
- 替代范围(全量 / 部分)
- 客户是否提供竞品资料(若提供,启动保密边界)
- 我方相对竞品的差异化(若明确)

### H 复购/补单

- 订单规格 / 数量 / 交期
- 历史表现(上次运行情况)
- 是否变更规格 / 工况 / 排放要求
- 包装 / 发票 / 交付地点
- 是否需要特殊检测报告

### I 有客户原话需整理

- 粘贴原文(触发后直接让用户粘贴)
- 不预设追问集,由原文决定任务类型

### J 其他/不确定

- 让用户从 A-H 选
- 若用户不能选,Round 1 退化为最小分流问题

## 任务-禁止承诺项 默认清单

每类任务在收敛后,本案专列禁止承诺项应至少包含:

- A 新项目: 不承诺寿命 / 不承诺排放 / 不锁单一型号
- B 故障: 不归因为材质 / 不承诺换料可解决 / 不承认不否认责任
- C 替换: 不承诺全量替代必成 / 不承诺价格优势 / 不承诺同寿命
- D 报价: 不出预算价外的承诺 / 不在无人工确认下报价 / 不承诺质保超规格
- E 试用: 不承诺试用必成功 / 不承诺试用期内零破损
- F 招投标: 不在无人工确认下盖章 / 不擅自承诺技术指标 / 不偏离招标条款底线
- G 竞品: 不评价竞品优劣 / 不引用竞品客户数据 / 不承诺替代率
- H 复购: 不承诺旧合同条款沿用 / 不擅自调整技术参数

## 任务-Block 触发条件

以下情况,无论任务类型,一律 Blocked:

- 客户/业务员要求"保证 X 年寿命 / 保证排放 ≤X mg / 全量替代 / 无试用 / 试用必成 / 直接换这个型号"。
- 责任归因压力(索赔、质量纠纷)且无证据。
- 客户原话缺失且拒绝补充。
- 商业/合规边界突破(招投标盖章、保密资料外泄、合同条款技术承诺)。
- 信息矛盾持续无法解决且拒绝澄清。

Blocked 后,显示 Blocked 状态 + 替代路径,不再继续技术追问。
