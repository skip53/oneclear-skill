# 22｜内部升级包

当输出可能被客户用于技术承诺、报价、试用、招投标、质保或合同附件时，需要内部升级包。

## 触发条件

- Level 4 候选方案。
- Level 5 技术支持包。
- 招投标、技术条款、偏差表、盖章承诺、合同附件。
- 排放、寿命、质保、完全替代、重大降本承诺。
- 安环风险、红色风险、No-Go 风险。
- 客户现场资料、竞品资料或检测报告涉及保密。

## 角色确认

| 角色 | 确认内容 |
| --- | --- |
| 技术负责人 | 候选方向、证据要求、风险假设、试用边界、禁止承诺项 |
| 销售负责人 | 决策链、预算阶段、采购流程、客户成功标准、时间线 |
| 管理层 | 招投标、质保、排放、重大商务让步或合同风险 |

## 升级包必须包含

- 客户需求摘要。
- 工况档案。
- 真实需求判断。
- 风险假设和风险升级等级。
- 候选方向及适用边界。
- 证据要求和缺口。
- 报价 readiness。
- 试用 / 验证边界。
- 客户话术草稿。
- 禁止承诺项。
- 下一步行动清单：负责人、需补资料、时间点、推进目标、目标 Level。

## 输出边界

升级包可以帮助内部决策，但不能自动变成对外承诺。对外版本必须删掉内部推断、竞品敏感信息和未经确认的结论。
