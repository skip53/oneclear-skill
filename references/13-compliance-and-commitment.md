# 13｜合规与承诺边界

工业滤料售前的"安全合规"主要不是软件数据安全，而是现场资料、环保排放、安环、质保、合同和招投标承诺边界。
本文件每类合规都按「**机制（为什么这条边界会触发） + 步骤化对策（怎么走流程） + 允许/禁止话术 + 升级条件**」四段式组织——读完一条，就知道**为什么这条边界必须守住**、**按什么顺序走流程**、**可以/不可以对客户说什么**、**什么情况下必须升级**。

> 通用原则：合规边界 = 法律 / 商业 / 安环 / 现场安全的硬线。**Agent 在边界内只输出"参考话术 + 必查证据清单"**——所有盖章 / 签字 / 合同 / 招投标承诺一律走人工三签（技术 + 销售 + 法务）。

---

## 环保排放

- **触发信号**：客户问"保证排放达标 / 超低排放 / 多少 mg/Nm³ / 多久能稳定达标"。

- **机制（为什么会触发）**：排放承诺 = 法律责任 + 监管风险。Agent 无现场试用的承诺 = 拿公司信誉对赌。**一旦工况与承诺条件不一致，触发环保处罚 + 客户索赔 + 监管连带**。即使后续补测达标，已发生的违约责任和监管记录不可逆。

- **步骤化对策**：
  1. **只记录目标 + 验证条件**——不保证具体数字
  2. **列必查证据**：当前排放曲线（CEMS）、工况稳定性、滤料选型适配、是否覆膜
  3. **走 red 升级**——任何"保证排放"请求必标 red
  4. **改用"目标话术"**：排放作为目标记录 + 是否能承诺需结合完整工况 + 样品/报告 + 试用结果 + 技术负责人确认

- **允许说什么**：「目前可以把排放作为目标记录，但能否承诺需要结合完整工况、检测报告、试用结果和技术负责人确认。」
- **禁止说什么**：「我们保证排放达标。」「能到 X mg/Nm³。」「一定能过环保验收。」
- **升级条件**：**必须升级**——任何"保证 / 一定 / 承诺" 类排放承诺 → 技术负责人 + 销售负责人 + 法务三签。

## 质保寿命

- **触发信号**：客户问"保用几年 / 保证寿命 / 同等寿命 / 不到寿命赔不赔"。

- **机制（为什么会触发）**：寿命承诺 = 客户选型决策的核心依据 + 后续索赔的法律起点。**工况与材料强度的耦合是非线性的**——同材料在不同温度/含湿/烟气成分下寿命可能差 3-5 倍。**没有现场试用 + 完整工况 + 技术评估就承诺寿命 = 拿未知条件签死合同**。

- **步骤化对策**：
  1. **只承诺同工况历史寿命范围**（不写具体数字到合同，参考 13 允许话术）
  2. **列必查证据**：同工况历史运行数据、材质匹配、工况稳定性
  3. **走 red 升级**——任何"保用 X 年 / 寿命保证"请求必标 red
  4. **改用"目标话术"**：寿命作为目标记录 + 是否能承诺需结合完整工况 + 同工况历史 + 技术负责人确认

- **允许说什么**：「目前可把 X 年寿命作为目标记录，但能否承诺需要结合完整工况、同工况历史、检测报告和技术负责人确认。」
- **禁止说什么**：「这个材质肯定能用两年。」「保用 18 个月。」「一定能用满寿命。」
- **升级条件**：**必须升级**——任何寿命数字 / 质保年限 / 不到寿命赔偿承诺 → 技术负责人 + 销售负责人 + 法务三签。

## 安环风险

- **触发信号**：客户问"火星 / 烧袋 / 爆炸 / 着火 / 安全事故 / 环保处罚"。

- **机制（为什么会触发）**：安环 = 不可逆事件。**一次火星烧穿 / 一次粉尘爆炸 = 人身伤害 + 监管处罚 + 永久关停风险 + 刑事责任**。滤料本身只能提供有限保护——火星源头在袋室之外，**单靠滤料无法解决安环问题**。如果 Agent 给出"换滤料即可解决安环"的承诺，会让客户把系统改造该做的火星捕集/阻火/喷淋/泄爆措施全部跳过。

- **步骤化对策**：
  1. **明确"滤料 + 系统措施" 双线**——滤料单独不能解决安环
  2. **列必查证据**：火星来源、阻火措施现状、峰值温度、安环现状
  3. **走 red 升级**——任何"换滤料解决火星"暗示必标 red
  4. **改用"系统话术"**：列系统措施清单（滤料 + 火星捕集 + 阻火 + 喷淋 + 泄爆），不只推滤料

- **允许说什么**：「火星/爆炸/烧袋需要系统措施——滤料 + 火星捕集 + 阻火 + 喷淋 + 泄爆，单靠滤料解决不了。请发安环现状与火星来源。」
- **禁止说什么**：「换 PTFE 滤袋就安全。」「能抗火星。」「着火找我们就行。」
- **升级条件**：**必须升级**——任何"换滤料即可安环"承诺或安环事件已发生 → 技术负责人 + 安环负责人 + 销售负责人。

## 客户资料保密

- **触发信号**：客户带现场照片 / 检测报告 / 运行记录 / DCS 导出数据。

- **机制（为什么会触发）**：客户资料 = 客户商业资产。**外传 = 违反保密协议 + 客户信任崩塌**。在售前阶段，业务员/Agent 经常"顺手把客户照片发朋友圈 / 群 / 同行"——这些行为**即使不构成法律诉讼，也触发客户信任崩塌 + 后续合作切断**。

- **步骤化对策**：
  1. **资料归档脱敏**——只作内部技术参考，不外传、不引用、不作对外承诺依据
  2. **标注使用范围**——哪些资料可引用、哪些不可
  3. **走 amber 升级**——涉及客户核心工艺 / 商业秘密的资料必标 amber
  4. **改用"内部话术"**：所有资料仅限内部技术判断，不与客户/同行/朋友圈分享

- **允许说什么**：「资料收到，仅作内部参考。所有判断基于我方产品在贵司工况下的表现。」
- **禁止说什么**：「这个客户/项目的照片我发个朋友圈。」「同行问起来我说一下。」「把客户数据脱敏后发个报告。」
- **升级条件**：业务员要求外传 / 客户要求书面记录 / 涉及客户商业秘密 → 销售负责人 + 法务。

## 竞品资料保密

- **触发信号**：客户带竞品出厂报告 / 检测数据 / PPT 资料 / 价格信息；业务员顺手评价竞品。

- **机制（为什么会触发）**：竞品资料 = 法律风险高发区。**评论竞品 = 留下书面证据**，后续可能用于反垄断诉讼 / 不正当竞争诉讼。**外传竞品资料 = 违反客户对前供应商的保密承诺**，触发客户对我方的信任崩塌。

- **步骤化对策**：
  1. **不评论竞品**——明确说"对竞品不做技术/价格/质量评价"
  2. **资料归档脱敏**——所有竞品资料仅作内部技术参考，不外传
  3. **回到我方工况**——所有判断回到"我方产品在客户工况下的表现"，不与竞品直接对比
  4. **走 red 升级**——业务员/客户要求书面评价竞品必标 red
- **允许说什么**：「资料收到，仅作内部参考。所有判断基于我方产品在贵司工况下的表现，不与竞品直接对比。」
- **禁止说什么**：「竞品确实不如…」「竞品价格更低…」「竞品有 XX 问题…」
- **升级条件**：业务员/客户要求书面评价竞品 → 销售负责人 + 法务。

## 招投标合同

- **触发信号**：客户问"招标 / 投标 / 技术响应 / 偏差表 / 盖章技术条款 / 合同附件"。

- **机制（为什么会触发）**：盖章条款 = 法律文件。**无证据承诺 → 合同条款写错 → 后续违约/纠纷**。招投标一旦废标 / 条款错 / 偏差表错，**补救成本极高**（可能直接废标 + 法律纠纷 + 失去该客户）。

- **步骤化对策**：
  1. **只输出"参考技术条款草案" + "未确定项清单"**——Agent 不直接出盖章条款
  2. **列必查证据**：完整工况、技术参数、偏差依据
  3. **走 red 升级**——任何盖章/合同/招投标请求必标 red
  4. **改用"草案话术"**：列参考技术条款 + 标注未确定项 + 由技术负责人 + 销售负责人 + 法务三签盖章
- **允许说什么**：「技术条款草案见附件，未确定项已标注。盖章版需技术 + 销售 + 法务三签后出具。」
- **禁止说什么**：「直接写进技术协议没问题。」「技术条款我们定了。」「招投标保证能过。」
- **升级条件**：**必须升级**——任何盖章 / 合同 / 招投标 / 偏差表 → 技术负责人 + 销售负责人 + 法务三签。

## 索赔责任

- **触发信号**：客户问"是不是滤料质量问题 / 能不能索赔 / 上家掺旧料吗 / 材质是不是不对"。

- **机制（为什么会触发）**：索赔 = 法律链路起点。**Agent 在证据链未建立前的归因 = 站队**——站客户一边（认质量问题）→ 后续供应商拒赔 → 客户反过来告 Agent 推荐失实；站供应商一边（否认问题）→ 客户失去信任。**两条路都输**。**不站队 = 把责任问题隔离到证据链后**。
- **步骤化对策**：
  1. **不承认不否认**——明确说"在证据链建立前，不做质量归因"
  2. **列证据需求**：同批是否同工况、是否有检测、是否有工况变化、是否有时序记录
  3. **走 red 升级**——任何索赔/质量责任判断必标 red
  4. **改用"证据话术"**：列证据链（送检 + 时序 + 工况对比）+ 转人工
- **允许说什么**：「责任判断需要证据链。请先发 ① 同批滤袋运行记录 ② 工况逐月变化 ③ 失效样品检测报告。」
- **禁止说什么**：「应该是…」「可能是上家…」「肯定不是我们的…」
- **升级条件**：**必须升级**——索赔/质量责任/责任归因 → 技术负责人 + 法务/销售负责人复核。

## 跨条目通用检查清单

> 任何合规边界触发时，先过这 4 条。

1. **是否触达法律 / 商业 / 安环硬线**？是 → 走 red 升级 + 三签流程。
2. **是否要求盖章 / 签字 / 合同**？是 → Agent 只出草案，不直接出盖章版。
3. **是否要求外传 / 公开**？是 → 不外传，回到我方工况。
4. **是否要求"保证 / 一定 / 承诺"**？是 → 用"目标 + 验证条件 + 人工确认"模板替代。
